Command Line Basics: Input & Output Environment

Your guide to essential terminal commands and navigation

Introduction to the Command Line

The command line interface (CLI) provides a powerful text-based way to interact with your computer's operating system. Understanding basic commands allows you to navigate, create, and manipulate files and directories efficiently.

This guide focuses on Unix-like systems (Linux, macOS). Windows users can use Windows Subsystem for Linux (WSL) or Git Bash to follow along.

user@hostname:~
$echo "Welcome to the command line!"
Welcome to the command line!

Basic Commands

Listing Files and Directories

The ls command displays the contents of the current directory:

ls
Terminal
$ls
Documents Downloads Music Pictures Videos

The ls command is one of the most frequently used commands for quickly checking directory contents.

Present Working Directory

The pwd command shows your current location in the file system:

pwd
Terminal
$pwd
/home/username

It's helpful to use pwd whenever you're unsure of your current location in the file system.

Clearing the Terminal

The clear command cleans up your terminal screen:

clear

Shortcut: You can also use Ctrl+L to clear the terminal in most systems.

Navigation

Change Directory (cd)

The cd command allows you to navigate between directories:

Moving into a directory:

cd Desktop
Terminal
username@hostname:~$cd Desktop
username@hostname:~/Desktop$

Moving out of the current directory (going back):

cd ..
Terminal
username@hostname:~/Desktop$cd ..
username@hostname:~$

Navigating multiple directories at once:

cd Desktop/folder1

Moving out multiple steps:

cd ../..

Use cd without any arguments to return to your home directory from anywhere.

Paths

Relative vs. Absolute Paths

Paths in the command line can be specified in two ways:

Relative Path (relative to current location):
Documents/Projects
Absolute Path (starts from root directory):
/home/username/Documents/Projects

Relative paths are specified relative to your current location.

Absolute paths always start with a forward slash (/) and specify the complete path from the root directory.

Special Path Symbols

Symbol Description Example
/ Root directory cd /
~ Home directory cd ~
. Current directory cd ./Documents
.. Parent directory cd ..

Moving to home directory:

cd ~

Moving to root directory:

cd /

The root directory (/) contains the entire file system, while your home directory (~) contains your personal files.

Creating Directories

Making Directories with mkdir

The mkdir command creates new directories:

Create a simple directory:

mkdir newDir
Terminal
username@hostname:~$mkdir newDir
username@hostname:~$ls
Documents Downloads Music newDir Pictures Videos

Create a directory at a specific path:

mkdir Desktop/folder1/newDir2

Create nested directories in one command:

mkdir -p Projects/website/css

The -p flag allows you to create parent directories if they don't exist.

Command Flags

Using Flags to Modify Commands

Flags are special options that modify how commands behave:

Terminal
username@hostname:~$ls -l
drwxr-xr-x 2 username group 4096 Mar 22 10:30 Documents drwxr-xr-x 2 username group 4096 Mar 21 15:45 Downloads drwxr-xr-x 2 username group 4096 Mar 20 09:15 Music drwxr-xr-x 2 username group 4096 Mar 22 11:20 newDir drwxr-xr-x 2 username group 4096 Mar 19 14:30 Pictures drwxr-xr-x 2 username group 4096 Mar 18 16:50 Videos

Commonly used flags with ls:

Flag Description Example
-l Long format with details about files ls -l
-a Show hidden files (starting with .) ls -a
-h Human-readable file sizes ls -lh
-R Recursive listing (includes subdirectories) ls -R

Flags can be combined: ls -la shows hidden files in long format.

Manual Pages

Getting Help with man

The man command displays the manual pages for commands:

man ls
Terminal
username@hostname:~$man ls
LS(1) User Commands LS(1) NAME ls - list directory contents SYNOPSIS ls [OPTION]... [FILE]... DESCRIPTION List information about the FILEs (the current directory by default). Sort entries alphabetically if none of -cftuvSUX nor --sort is specified. Mandatory arguments to long options are mandatory for short options too. -a, --all do not ignore entries starting with . -l use a long listing format ... Manual page ls(1) line 1

Navigate manual pages using arrow keys, space (next page), b (back), and q (quit).

If you just want a brief summary of a command, try command --help or command -h.

Creating Files

Using touch to Create Files

The touch command creates empty files or updates file timestamps:

touch index.html
Terminal
username@hostname:~$touch index.html
username@hostname:~$ls
Documents Downloads index.html Music newDir Pictures Videos

Create multiple files at once:

touch style.css script.js README.md

While touch is commonly used to create new files, its original purpose is to update the access and modification timestamps of existing files.

Deleting Files and Directories

Removing Files with rm

The rm command removes files:

rm filename.txt
Terminal
username@hostname:~$touch temp.txt
username@hostname:~$ls
Documents Downloads index.html Music newDir Pictures temp.txt Videos
username@hostname:~$rm temp.txt
username@hostname:~$ls
Documents Downloads index.html Music newDir Pictures Videos

Removing Empty Directories with rmdir

The rmdir command removes empty directories:

rmdir emptyFolder

Removing Directories with rm -rf

To remove non-empty directories and their contents:

rm -rf directoryName

WARNING: The rm -rf command is extremely powerful and potentially dangerous. It will delete directories and all their contents without confirmation. Files deleted this way cannot be recovered from a trash bin.

Be especially careful when using rm -rf with wildcards or as the root user.

Use rm -i for interactive mode, which will prompt for confirmation before deleting each file.

Best Practices

Command Line Best Practices

  • Use tab completion to avoid typing errors and save time
  • Be cautious with destructive commands like rm -rf
  • Learn to use relative and absolute paths appropriately
  • Make use of command history (up arrow) to recall previously used commands
  • Add meaningful comments to your command line scripts using #
  • Use clear and descriptive names for files and directories

Input and Output Environment

Standard Streams

The command line operates with three standard streams:

  • stdin (0): Standard input - data fed into the program
  • stdout (1): Standard output - data printed by the program
  • stderr (2): Standard error - error messages generated by the program

Redirection

You can redirect output to files instead of displaying it on the screen:

Output Redirection:

ls > files.txt

Redirects ls output to files.txt (overwrites)

ls >> files.txt

Appends ls output to files.txt

Terminal
username@hostname:~$ls > files.txt
username@hostname:~$cat files.txt
Documents Downloads index.html Music newDir Pictures Videos

Input Redirection:

sort < data.txt

Uses data.txt as input for the sort command

Error Redirection:

command 2> errors.log

Redirects error messages to errors.log

Combined Redirection:

command > output.txt 2> errors.log

Separate files for output and errors

command > all.log 2>&1

Both output and errors to same file

Redirection is one of the most powerful features of the command line, allowing you to save, process, and manage command outputs efficiently.

Pipes

Pipes (|) allow you to use the output of one command as the input for another:

ls -l | grep "txt"

Lists only files containing "txt"

Terminal
username@hostname:~$ls -l | grep "txt"
-rw-r--r-- 1 username group 0 Mar 22 12:35 files.txt
cat file.txt | sort

Sorts the contents of file.txt

ps aux | grep firefox

Finds firefox processes

You can chain multiple pipes together: command1 | command2 | command3

Command Substitution

You can use the output of a command as part of another command:

echo "Today is $(date)"

Embeds the output of date command

Terminal
username@hostname:~$echo "Today is $(date)"
Today is Sat Mar 22 14:45:23 EDT 2025
cp file.txt backup-$(date +%Y%m%d).txt

Filename with today's date

Environment Variables

Environment variables store information about your command line environment:

echo $HOME

Displays home directory path

Terminal
username@hostname:~$echo $HOME
/home/username
echo $PATH

Shows executable search paths

echo $USER

Shows current username

Setting Variables:

export NAME="value"

Sets environment variable

NAME="value"

Sets local variable

Command Line Text Processing

cat - Concatenate and Display Files

cat file.txt

Display file contents

cat file1.txt file2.txt > combined.txt

Combine files

grep - Search Text

grep "pattern" file.txt

Find lines containing "pattern"

grep -i "pattern" file.txt

Case-insensitive search

grep -r "pattern" directory

Recursive search in directory

Terminal
username@hostname:~$echo "Hello World" > greeting.txt
username@hostname:~$grep "Hello" greeting.txt
Hello World

head and tail - View File Parts

head -n 10 file.txt

Show first 10 lines

tail -n 5 file.txt

Show last 5 lines

tail -f log.txt

Follow file updates in real-time

tail -f is extremely useful for monitoring log files as they're being written to.

wc - Count Words, Lines, Characters

wc file.txt

Count lines, words, bytes

wc -l file.txt

Count lines only

Terminal
username@hostname:~$wc greeting.txt
1 2 12 greeting.txt
username@hostname:~$wc -l greeting.txt
1 greeting.txt

Command History and Completion

Command History

history

Show command history

Terminal
username@hostname:~$history
1 ls 2 cd Documents 3 pwd 4 cd .. 5 echo "Hello World" > greeting.txt 6 grep "Hello" greeting.txt
!42

Run command number 42 from history

!!

Run the previous command

Tab Completion

Press Tab to:

  • Complete file and directory names
  • Complete command names
  • Complete command options (in some shells)

Tab completion is one of the most time-saving features of the command line. Use it frequently!

Command Chaining

command1 && command2

Run command2 only if command1 succeeds

command1 || command2

Run command2 only if command1 fails

command1 ; command2

Run command2 regardless of command1's outcome

Terminal
username@hostname:~$mkdir project && cd project
username@hostname:~/project$

Command chaining helps you create more complex operations without writing scripts.

Advanced Topics

File Permissions

chmod 755 script.sh

Set read, write, execute permissions

chmod +x script.sh

Add execute permission

chown user:group file

Change file ownership

Process Management

ps

List current processes

ps aux

List all processes in detail

kill PID

Terminate process by ID

killall firefox

Terminate processes by name

top

Monitor processes in real-time

Terminal
username@hostname:~$ps
PID TTY TIME CMD 1234 pts/0 00:00:01 bash 5678 pts/0 00:00:00 ps

Searching Files

find . -name "*.txt"

Find .txt files in current directory

find /home -type d -name "Downloads"

Find directories named Downloads

find . -type f -mtime -7

Find files modified in last 7 days

Terminal
username@hostname:~$find . -name "*.txt"
./files.txt ./greeting.txt

The find command is extremely powerful with many options. Check man find for all possibilities.

The command line is a powerful interface that allows direct interaction with your computer's operating system. Mastering basic commands and understanding input/output redirection, pipes, and environment variables will significantly enhance your productivity and technical capabilities.

Remember that practice is essential - the more you use the command line, the more comfortable and proficient you'll become.
